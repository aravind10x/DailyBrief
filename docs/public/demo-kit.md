# Demo Kit

This doc packages the repo for public-facing screenshots and a clean GitHub presentation.

## Narrative

The story to tell is simple:

1. A founder starts with the real context of the company.
2. Daily Brief turns that context into a clear operating plan.
3. The founder approves the plan and moves into execution.
4. The system learns from what moved and what got deferred.

Keep every screenshot in service of that story.

## Screenshot Set

Save final assets under `docs/public/assets/screenshots/`.

## Capture Checklist

- Use a desktop viewport around `1440x1024`.
- Keep browser zoom at `100%`.
- Hide bookmarks bars, devtools, and browser clutter where possible.
- Prefer the seeded `AstraPilot` flow for all public assets.
- Use the same browser and window size for every shot.
- Capture after approving the plan for execution assets.
- Avoid showing localhost URL bars if you can crop cleanly.
